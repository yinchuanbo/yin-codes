  "01 原生嵌套CSS特性.md": "<h1>2024 年新的原生嵌套 CSS 特性</h1>\n<p>新的原生嵌套 CSS 特性彻底改变了前端开发。</p>\n<p>❌ 之前：</p>\n<p>你会如何为这些嵌套的 HTML 元素添加样式？</p>\n<pre><code class=\"language-html\">&lt;section&gt;\n  你好！\n  &lt;div&gt;\n    &lt;p&gt;&lt;span&gt;codingbeautydev.com&lt;/span&gt; -- 以编码为激情&lt;/p&gt;\n    &lt;p&gt;编码帮助你实现目的感和成长。&lt;/p&gt;\n  &lt;/div&gt;\n&lt;/section&gt;\n</code></pre>\n<p>你通常会感到压力，浪费大量时间重复外部元素名称。</p>\n<pre><code class=\"language-css\">section {\n  font-family: Arial;\n}\n\nsection div {\n  font-size: 1.5em;\n}\n\nsection div p {\n  color: blue;\n}\n\nsection div p span {\n  font-weight: bold;\n}\n</code></pre>\n<p>难怪 SASS 和 LESS 变得如此受欢迎。</p>\n<p>✅ 但现在：使用原生 CSS：</p>\n<p>变得更加清晰和简单。所有样式现在都被封装在一起，而不是散落各处。</p>\n<pre><code class=\"language-css\">section {\n  font-family: Arial;\n  div {\n    font-size: 1.2em;\n    p {\n      color: blue;\n      span {\n        font-weight: bold;\n      }\n    }\n  }\n}\n</code></pre>\n<p>就像面向对象编程中的封装一样直观：</p>\n<pre><code class=\"language-js\">// ❌ redundancy\n// ❌ 冗余\nconst personName = &quot;Tari Ibaba&quot;;\nconst personSite = &quot;codingbeautydev.com&quot;;\nconst personColor = &quot;🔵blue&quot;;\n\n// ✅ encapsulate field\n// ✅ 封装字段\nclass Person {\n  name = &quot;Tari Ibaba&quot;;\n  site = &quot;codingbeautydev.com&quot;;\n  color = &quot;🔵blue&quot;;\n}\n</code></pre>\n<p>在某些浏览器中，你可能需要使用 <code>&amp;</code>：</p>\n<pre><code class=\"language-css\">section {\n  font-family: Arial;\n  &amp; div {\n    font-size: 1.2em;\n    &amp; p {\n      color: blue;\n      &amp; span {\n        font-weight: bold;\n      }\n    }\n  }\n}\n</code></pre>\n<h2>那么类和 ID 呢？</h2>\n<p>如果你想通过 <code>class</code> 或 <code>id</code> 属性来为嵌套元素添加样式怎么办？</p>\n<pre><code class=\"language-html\">&lt;section class=&quot;class1&quot;&gt;\n  你好！\n  &lt;div id=&quot;id1&quot;&gt;\n    &lt;p class=&quot;class2&quot;&gt;\n      &lt;span id=&quot;url&quot;&gt;codingbeautydev.com&lt;/span&gt; -- 以编码为激情\n    &lt;/p&gt;\n    &lt;p&gt;编码具有认知挑战性和精神刺激性。&lt;/p&gt;\n  &lt;/div&gt;\n&lt;/section&gt;\n</code></pre>\n<p>嵌套 CSS 将非常相似：</p>\n<pre><code class=\"language-css\">.class {\n  font-family: Arial;\n  #id1 {\n    font-size: 1.2em;\n    .class2 {\n      color: purple;\n      #url {\n        font-weight: bold;\n      }\n    }\n  }\n}\n</code></pre>\n<p>它也适用于兄弟选择器：</p>\n<pre><code class=\"language-css\">div {\n  section {\n    + p {\n      color: blue;\n    }\n    ~ p {\n      color: red;\n    }\n  }\n}\n</code></pre>\n<h2>伪类和伪元素</h2>\n<p>是的：</p>\n<pre><code class=\"language-css\">button {\n  background-color: blue;\n  :hover {\n    background-color: yellow;\n  }\n}\n</code></pre>\n<h2>媒体查询</h2>\n<p>嵌套 CSS 的另一个巨大卖点：</p>\n<p>❌ 之前：</p>\n<p>创建媒体查询很混乱，查询样式和元素的主要样式是分开的：</p>\n<pre><code class=\"language-css\">.hamburger {\n  display: none;\n}\n\n.header {\n  font-size: 40px;\n}\n\n@media (orientation: landscape) {\n  .header {\n    font-size: 32px;\n  }\n}\n\n@media (max-width: 480px) {\n  .hamburger {\n    display: block;\n  }\n  .header {\n    font-size: 24px;\n  }\n}\n</code></pre>\n<p>✅ 现在：</p>\n<p>让元素样式包含查询样式比让查询样式包含元素样式的小片段更直观。</p>\n<p>嵌套 CSS 让你可以轻松做到这一点：</p>\n<pre><code class=\"language-css\">.hamburger {\n  display: none;\n\n  @media (max-width: 480px) {\n    display: block;\n  }\n}\n\n.header {\n  font-size: 40px;\n\n  @media (orientation: landscape) {\n    font-size: 32px;\n  }\n\n  @media (max-width: 480px) {\n    font-size: 24px;\n  }\n}\n</code></pre>\n<p>使用原生嵌套 CSS，你可以更直观地创建样式。</p>\n<p>SASS 现在基本上没用了 — 特别是现在我们在 CSS 中也有了原生变量：</p>\n<pre><code class=\"language-css\">$base-font-size: 16px;\n$gutter-width: 10px;\n\n.container {\n  padding: calc($gutter-width * 2);\n  font-size: $base-font-size;\n}\n\n.heading {\n  font-size: calc($base-font-size * 1.5);\n}\n</code></pre>\n"
